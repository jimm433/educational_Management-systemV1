【區域一：惡意攻擊分類】
--------------------------------------------------------------------------------
【惡意攻擊分類】\n- 主動注入 (Active Injections)：這類攻擊指的是攻擊者將惡意提示或指令主動傳遞給一個整合了 LLM 功能的應用程式或系統，使得 LLM 在處理該輸入時，執行非預期的操作或產生惡意輸出。這種方式的目的是竊取敏感資料、提供非預期的輸出，或誘騙 LLM 運行惡意提示。\n- 被動注入(Passive Injections)：這類攻擊涉及攻擊者在大型語言模型可能讀取的公開資料來源中植入惡意提示或內容。當 LLM 讀取這些被操縱的資料（例如透過外掛程式或其他集成方式）時，它會無意中執行這些嵌入的指令。這種方法旨在誘騙 LLM 提供錯誤資訊或執行惡意提示。\n- 使用者驅動注入(User-driven Injections)：這類攻擊利用社交工程技巧，向毫無戒心的使用者提供看似無害的提示。使用者在不知情的情況下，將這些提示複製並輸入到 LLM 中執行，進而導致 LLM 執行攻擊者預設的惡意行為。
    ◦ 這是對 LLM 一般終端使用者而言最主要的威脅之一。\n- 虛擬提示注入(Virtual Prompt Injection)：這類攻擊依賴於攻擊者能夠在大型語言模型進行指令微調（instruction tuning）的訓練階段，對訓練資料進行篡改或惡意植入。其目的是在 LLM 部署到生產環境後，在特定情境下導致模型產生偏離、被操縱的行為或輸出，無需在運行時直接操控提示。即使少量的受污染訓練範例，也可能顯著扭曲模型在特定主題上的輸出。\n- 雙重角色(Double Character)：這是一種提示策略，旨在誘導 LLM 生成兩種不同的回應：一種遵循其標準的安全規範，另一種則繞過內容限制，以不受約束的方式生成惡意內容或不當輸出。攻擊者透過建立一種「雙重角色」的情境，使 LLM 在模擬其中一個角色時無視其預設的安全過濾機制。\n- 虛擬化(Virtualization)：此類攻擊透過將 LLM 置於一個模擬的、不受限制的模式或虛擬情境中（例如「開發者模式」或「虛擬機器」內），誘使其在該虛擬環境中生成惡意或受限內容。攻擊者透過創造一個虛構的環境，聲稱在該環境中，LLM 的行為不受常規規則的約束，從而繞過其安全協議。\n- 模糊處理(Obfuscation)：這種攻擊涉及對提示中的惡意內容或違反規則的指令進行模糊處理，使其不被 LLM 的輸入或輸出過濾器直接識別。這可能包括使用編碼（例如 Base64 字元）、同義詞或拼寫錯誤等方式，以隱藏真實意圖並規避安全檢測機制。\n- 負載拆分(Payload Splitting)：這是一種將惡意指令拆分為多個獨立的、單獨看來無害的提示，然後在最終提示中將這些組件組合起來，從而觸發惡意行為。每個單獨的提示可能不會觸發安全警報，但當 LLM 處理並整合所有提示後，其結合的意圖即會顯現為惡意。\n- 對抗性後綴(Adversarial Suffix)：這類攻擊涉及將一段計算機生成且對人類而言無意義的字詞和字元串（後綴）附加到惡意提示上。這個後綴旨在規避 LLM 的對齊（alignment）機制，使其在回應惡意提示時產生不當輸出，而無需手動實驗來製作提示。\n- 指令操縱(Instruction Manipulation)：此類攻擊旨在揭露 LLM 介面預先設定的內部指令或「初始提示」，或者直接指示 LLM 忽略這些既定指令。透過獲取這些隱藏的設定資訊，攻擊者可以更有效地設計提示來繞過模型的限制，或直接命令模型無視其安全協議。
